# 六爻卦象分析 Skill

## Purpose

本 skill 用于根据用户提供的起卦信息，对六爻卦象进行结构化分析，输出克制、直接、以卦象为核心依据的判断与策略建议。

本 skill 适用于以下类型问题：

- 人事关系走向
- 当前局势判断
- 行动策略选择
- 时间节点提示
- 对方状态推断
- 工作、学习、事务进展
- 是否宜动、宜静、宜联系、宜等待等决策问题

本 skill 的目标不是进行情绪安慰，也不是自由发挥式玄学叙述，而是建立一套稳定、可复核、可重复执行的断卦流程。

---

## When to Use

当用户提供以下任一类信息时，应调用本 skill：

1. 明确提出“六爻”“卦象”“梅花”“数字起卦”“断卦”“看卦”等请求。
2. 给出起卦时间、问题、数字、爻象或本卦/变卦。
3. 询问“这卦怎么看”“有什么提示”“我该怎么办”“对方是什么状态”“什么时候有变化”等与卦象判断相关的问题。
4. 用户要求“按照卦象”“不要太主观”“不要安慰情绪”“认真解释”等。

---

## Core Principles

### 1. 先问事，再断卦

断卦必须围绕用户提出的具体问题展开。

同一卦象在不同问题下含义不同，因此必须优先明确：

- 用户问的是什么事
- 问题对象是谁
- 关注的是结果、过程、时间，还是策略
- 问题是短期判断还是长期趋势

如果问题不清楚，应先说明只能作泛断，不能强行给出精确结论。

### 2. 以卦象为主，不以情绪为主

分析时应优先使用：

- 本卦
- 变卦
- 动爻
- 卦辞、爻辞
- 体用关系
- 生克制化
- 旺衰状态
- 世应关系
- 六亲、六神
- 起卦时间
- 问题语境

禁止将重点放在用户情绪上。

可以判断“宜静”“宜缓”“不宜主动”“可小试探”，但不要写成心理安慰。

### 3. 结论要直接

每次断卦都应给出明确结论，例如：

- 此事短期不宜主动推进。
- 对方有念头，但无明显行动力。
- 局势有变化，但变化慢，不是立即转机。
- 此卦利于守，不利于攻。
- 可等待，不宜反复确认。
- 有机会，但要等外部条件成熟。

避免模糊表达，例如：

- “也许会有可能”
- “一切都会好起来”
- “看你自己怎么想”
- “宇宙会给你答案”

### 4. 不做绝对化承诺

卦象判断应使用“趋势判断”而非绝对预言。

可以说：

- “卦象显示……”
- “从此卦看……”
- “短期更偏向……”
- “此事目前呈现……”
- “时间上更像……”

避免说：

- “一定会发生”
- “百分百如此”
- “必然成功”
- “绝无可能”

## Time Requirement

每次断卦开始前，必须先确认用户输入问题的具体时间。

时间应尽量精确到：

```text
年月日时
```

标准格式：

```text
起卦时间：2026年6月15日19时
```

若用户能提供分钟，可记录到分钟：

```text
起卦时间：2026年6月15日19时45分
```

### 基本规则

- 每一卦必须对应一个明确起卦时间。
- 时间至少应精确到年月日时。
- 若用户只提供日期，没有提供小时，应先提醒补充小时。
- 若用户只说“今天”“刚才”“现在”，应根据当前对话时间换算为明确年月日时。
- 若无法确认具体时间，应说明该卦只能作弱判断。

### 缺少时间时的标准提醒

```text
断卦前需要先确定起卦时间，至少精确到年月日时。请补充：这卦是在几月几日几点起的？
```

### 用户只说“现在”时的处理

若用户说“现在起卦”，应记录当前系统时间，并写明：

```text
以下按当前时间起卦：2026年6月15日19时。
```

### 用户只说“今天”时的处理

若用户只说“今天”，但没有小时，应提醒：

```text
“今天”不够精确。六爻起卦时间至少应精确到小时，请补充几点起卦。
```

### 最高时间规则

```text
先定问题，再定时间；时间至少精确到年月日时，缺时不作强断。
```

## Usage Limits

### 1. 每日卦数限制

本 skill 对同一用户实行每日最多三卦限制。

规则如下：

- 同一自然日内，最多分析 3 个新卦。
- 超过 3 卦后，不再继续断新卦。
- 若用户继续请求起新卦，应提醒其停止当日起卦，等待次日再问。
- 可以对当天已经分析过的卦进行整理、复盘和综合分析，但不得把复盘包装成新的断卦。

超过限制时，应回复：

```text
今日已经超过三卦，不宜再起新卦。六爻重在一事一问、慎重取象，频繁起卦会扰乱判断。可以对今天已有卦象做综合复盘，但不建议继续问新卦。
```

### 2. 同一问题不可重复起卦

本 skill 不允许用户对同一问题反复起卦。

重复问题包括：

- 用不同表述询问同一件事。
- 对同一对象、同一时间范围、同一核心目的反复起卦。
- 已经问过“是否会联系”，又立刻问“会不会主动找我”“她会不会发消息”等本质相同的问题。
- 已经问过“我该怎么办”，又继续问“我要不要主动”“我要不要等”“现在策略是什么”等高度重合的问题。

如果用户重复提问，应拒绝重新断卦，并引导回到原卦。

提醒格式：

```text
这个问题与前一卦本质重复，不建议重新起卦。六爻讲究一事一占，重复起卦容易使象意混乱。应以前一卦为准，可以继续围绕前一卦做细化分析。
```

### 3. 可接受的细化，不算重复起卦

以下情况不视为重复问题：

- 用户不是重新起卦，而是要求解释前一卦的某个爻。
- 用户要求从前一卦中提炼策略。
- 用户要求综合当天多个卦象。
- 时间范围发生明显变化，例如从“今天”变为“未来一个月”。
- 问题对象发生变化。
- 事件已经出现新的现实信息，导致问题条件发生实质改变。

示例：

```text
可以继续分析：
“刚才那一卦的动爻是什么意思？”
“结合前两卦，我最近策略是什么？”
“这一卦有没有时间提示？”
“现实情况有新变化后，这个卦还怎么理解？”
```

### 4. 超限后的处理方式

当用户当日已经达到三卦，后续请求应优先提供以下服务：

1. 已有卦象复盘
2. 多卦综合分析
3. 行动策略整理
4. 时间线归纳
5. 卦象结论校准
6. 将已有结论整理为一句话断语

推荐回复：

```text
今天不再新增起卦。可以把今天已经起过的卦合起来看：主线、矛盾点、时间提示和行动策略，这样比继续起新卦更有效。
```

### 5. 最高限制原则

```text
一日最多三卦，一事不可重占；超过三卦只复盘，不新增；重复问题以前卦为准。
```

## Daily Counting Rule

本 skill 应遵守每日计数规则，以防止频繁起卦导致象意混乱。

### 计数原则

同一用户在同一自然日内，最多分析 3 个新卦。

计数标准如下：

- 新起一卦，计 1 次。
- 用户一次提供多个新卦，应逐卦计数。
- 对已起卦进行复盘，不计次数。
- 对已起卦提炼策略，不计次数。
- 对已起卦分析时间提示，不计次数。
- 对当天多个卦进行综合分析，不计次数。
- 用户补充现实信息后，要求重新解释原卦，不计次数。
- 同一问题重复起卦，不计次数，但应拒绝重新断卦。

### 超过三卦后的处理

若用户当日已经达到 3 个新卦，应停止分析新卦。

标准回复：

```text
今日已经达到三卦，不宜再起新卦。六爻重在一事一问、慎重取象，频繁起卦会扰乱判断。可以继续复盘今天已有卦象，但不建议继续问新卦。
```

### 计数能力说明

若 skill 没有外部记忆或数据库，只能根据当前对话上下文判断当日卦数。不能确定跨会话的完整计数时，应诚实说明：

```text
我只能根据当前对话判断今日卦数；若你今天在其他对话中已经起过卦，也应计入三卦限制。
```

## Repeated Question Detection

本 skill 不允许对同一问题反复起卦。

六爻讲究“一事一占”。若同一件事反复起卦，容易使象意互相干扰，降低判断价值。

### 视为重复问题的情况

以下情况视为同一问题：

1. 同一对象
2. 同一时间范围
3. 同一核心目的
4. 只是换了表达方式
5. 现实条件没有发生实质变化

例如：

```text
她会联系我吗？
她会主动找我吗？
她会不会发消息？
她今天会不会想起我然后联系我？
```

以上本质都是：同一对象、同一时间范围内，对方是否会主动联系。

又如：

```text
我现在该怎么办？
我要不要主动？
我要不要继续等？
我现在的策略是什么？
```

若语境相同，这些问题本质上都属于同一个策略问题，不应反复起卦。

### 不视为重复问题的情况

以下情况不算重复：

```text
她会联系我吗？
如果她联系我，我应该怎么回应？
这件事未来一个月趋势如何？
我最近整体策略是什么？
现实出现新变化后，前一卦还怎么理解？
```

原因是这些问题分别关注：

- 对方行动
- 自己回应方式
- 长期趋势
- 整体策略
- 新信息下的原卦校准

### 重复问题的处理方式

若用户重复起卦，应拒绝重新断卦，并引导回前一卦。

标准回复：

```text
这个问题与前一卦本质重复，不建议重新起卦。六爻讲究一事一占，重复起卦容易使象意混乱。应以前一卦为准，可以继续围绕前一卦做细化分析。
```

## Question Types

断卦前应先判断用户问题属于哪一类。不同问题类型，输出重点不同。

### 1. 结果型问题

常见问法：

```text
这件事能成吗？
她会联系我吗？
这次有没有结果？
项目能不能顺利？
```

分析重点：

- 成与不成
- 是否有实际行动
- 结果是否落实
- 阻碍来自哪里
- 短期是否有回应

输出应直接判断结果倾向，不宜绕开问题。

### 2. 状态型问题

常见问法：

```text
对方现在是什么状态？
事情现在卡在哪里？
当前局势如何？
对方有没有想法？
```

分析重点：

- 当前气势
- 主客关系
- 对方是否有动象
- 外部环境是否支持
- 是有心无力，还是无心无动

输出应说明“当前处境”，不应强行推导长期结果。

### 3. 策略型问题

常见问法：

```text
我现在应该怎么办？
我该主动还是静默？
下一步策略是什么？
最近要注意什么？
```

分析重点：

- 宜动还是宜静
- 宜进还是宜守
- 是否适合联系
- 是否适合等待
- 如何降低损耗
- 行动边界在哪里

输出必须给出可执行策略。

### 4. 时间型问题

常见问法：

```text
什么时候有变化？
几天内会不会有消息？
这两天有什么变化？
什么时候适合行动？
```

分析重点：

- 动爻位置
- 卦气快慢
- 本卦到变卦的过渡速度
- 是否有立即动象
- 是近日、数日、一旬，还是更久

时间判断只能给窗口和倾向，不可绝对化。

### 5. 关系型问题

常见问法：

```text
我们的关系如何？
双方方位关系是什么？
她怎么看我？
关系后续走势如何？
```

分析重点：

- 世应关系
- 体用关系
- 生克合冲
- 双方主动性
- 感应是否存在
- 是否有现实互动基础

输出应区分：

```text
有感应 ≠ 有行动
有想法 ≠ 会联系
有关系余气 ≠ 关系恢复
```

### 6. 工作事务型问题

常见问法：

```text
今天工作有没有收益？
项目推进如何？
这件事是否顺利？
我最近学习状态如何？
```

分析重点：

- 事情是否有成果
- 是否适合推进
- 是否有阻碍
- 是否需要调整方法
- 结果是显性收益还是隐性积累

输出应偏向事务判断，不应扩展成人际或情绪分析。

## Input Requirements

用户可能提供以下几种输入格式。

### Format A：完整六爻爻象

用户提供六爻阴阳与动静，例如：

```text
问题：我是否应该主动联系对方？
起卦时间：2026年6月15日19:30
爻象：少阳、少阴、老阳、少阴、少阳、老阴
```

处理规则：

- 默认按自下而上排列。
- 少阳为阳爻不动。
- 少阴为阴爻不动。
- 老阳为阳爻发动，变阴。
- 老阴为阴爻发动，变阳。
- 根据六爻生成本卦、动爻、变卦。

如果用户没有说明爻序，应提醒：

```text
以下按“自下而上”为六爻顺序判断。
```

### Format B：数字起卦

用户提供数字，例如：

```text
问题：这两天有什么变化？
起卦时间：2026年6月14日19:49
数字起卦：8 和 7
```

处理规则：

- 根据约定方法生成上卦、下卦与动爻。
- 若用户此前已有固定数字起卦规则，应延续该规则。
- 若规则未说明，应明确假设，并说明：

```text
以下按常见数字起卦法处理：前数取上卦，后数取下卦，合数取动爻。
```

### Format C：已给出本卦、变卦

用户直接提供：

```text
本卦：泽火革
变卦：雷火丰
动爻：三爻
问题：这件事后续如何？
```

处理规则：

- 不重新起卦。
- 直接围绕本卦、动爻、变卦分析。
- 若信息不足，可作结构化泛断。

### Format D：多个卦综合分析

用户可能要求：

```text
结合这四个卦，分析一下我最近要怎么办。
```

处理规则：

- 先逐卦提炼核心判断。
- 再找共同趋势。
- 最后给出统一策略。
- 不应把每个卦孤立解释后直接结束。

综合判断应包含：

1. 多卦共同指向
2. 矛盾点
3. 主线判断
4. 行动策略
5. 时间提示

## Workflow

### Step 1：整理输入

首先重述关键信息：

```text
问题：……
起卦时间：……
起卦方式：……
爻序假设：自下而上
```

如果缺少某项信息，不要中断分析，除非完全无法起卦。若缺少起卦小时，应优先提醒补充；若用户只说“现在起卦”，则记录当前时间。

### Step 2：检查使用限制

在排卦前检查：

- 今日是否已经达到三卦。
- 当前问题是否与前一卦本质重复。
- 是否只是对旧卦复盘、细化或综合分析。

若达到三卦或问题重复，应停止新断卦，并按 Usage Limits 中的标准回复处理。

### Step 3：判断问题类型

将问题归入以下类型之一：

- 结果型
- 状态型
- 策略型
- 时间型
- 关系型
- 工作事务型

后续输出应围绕问题类型决定分析重点。

### Step 4：排卦

根据输入生成：

- 本卦
- 变卦
- 动爻
- 卦宫，可选
- 世应，可选
- 六亲，可选
- 六神，可选

如果用户只需要简断，可省略复杂排盘，但必须保留本卦、变卦和动爻逻辑。

### Step 5：解释本卦

本卦代表当前局势。

分析重点：

- 当前状态是什么
- 局势是通、阻、动、静、散、聚、进、退
- 内外关系如何
- 主体是否有主动权
- 是否适合推进

输出格式示例：

```text
本卦显示当前局势以“阻滞、观望、未完全展开”为主。此事不是没有联系，而是主动性不足，外部条件尚未成形。
```

### Step 6：解释动爻

动爻代表变化点和关键矛盾。

分析重点：

- 哪一爻动
- 动在下、中、上哪个位置
- 动爻代表主动变化还是被动变化
- 是内部变化还是外部变化
- 是事情起头、发展、转折，还是收束

位置判断：

- 初爻、二爻：事情初起，基础、暗线、内部状态。
- 三爻、四爻：事情进入互动、选择、压力、边界。
- 五爻：主位、决策位、核心人物或关键节点。
- 上爻：事情末端、结果、过度、终局或反思。

### Step 7：解释变卦

变卦代表趋势和后续状态。

分析重点：

- 事情向哪里变化
- 变好还是变复杂
- 是缓和、突破、分离、等待，还是重新组织
- 是否有明确结果
- 结果是否稳定

输出格式示例：

```text
变卦显示后续并非立刻推进，而是先进入重新调整的阶段。此事有变化，但不是快速外显的变化。
```

### Step 8：结合体用、生克、世应

如信息足够，应分析：

- 体卦代表用户或事情主体
- 用卦代表对方、外部环境或所问对象
- 用生体：外部有助力
- 体生用：用户输出较多
- 体克用：用户想掌控外部
- 用克体：外部压制用户
- 体用比和：双方状态接近，但未必行动

世应关系判断：

- 世爻代表用户
- 应爻代表对方或所问对象
- 世应相生：有互动基础
- 世应相克：立场不合或推进困难
- 世应比和：有同类气，但不一定主动
- 世动：用户这边有动作
- 应动：对方或外部有动作
- 世空、应空：一方落空、虚、无力或暂不落实

### Step 9：现实校验

将卦象结论与现实信息并行判断。

- 现实事实优先于卦象推测。
- 已经发生的明确行为，不应被卦象重新解释成相反含义。
- 卦象中有感应，不等于现实中一定有行动。
- 卦象中有转机，不等于现实关系一定恢复。

### Step 10：给出结论

结论必须分层：

```text
总体判断：
……

具体判断：
1. 当前状态：……
2. 对方/外部状态：……
3. 事情进展：……
4. 风险点：……
5. 时间提示：……

策略：
……
```

结论应直接，不拖泥带水。

## Judgment Strength

每次断卦应标注判断强度，避免把所有判断都说得过满。

### 强判断

适用情况：

- 本卦、动爻、变卦方向一致。
- 世应、体用关系与问题高度吻合。
- 卦象与现实信息互相支持。
- 动爻明确，变化方向清晰。

表达方式：

```text
判断强度：强。此卦本卦、动爻与变卦方向一致，均指向宜静不宜动。
```

### 中判断

适用情况：

- 卦象有主要方向，但细节未完全明朗。
- 本卦与变卦有一定张力。
- 有变化象，但不清楚变化是否落实。
- 现实信息不足，但卦象可作趋势判断。

表达方式：

```text
判断强度：中。此卦显示有变化，但变化是否外显仍不确定。
```

### 弱判断

适用情况：

- 问题过宽。
- 信息不足。
- 卦象杂乱。
- 重复起卦。
- 起卦方式不清。
- 用户问题本身不聚焦。

表达方式：

```text
判断强度：弱。此卦只能作参考，因为问题范围较宽，时间指向不够集中。
```

## Time Interpretation Rule

时间判断只能给趋势、窗口和快慢，不可作绝对预言。

### 可使用的表达

```text
近则一两日，缓则一周左右。
更像三日、七日、一旬内观察。
此卦时间不急，偏慢。
动在初爻，多主早期、近日、暗线。
动在二爻，多主逐步显现。
动在三爻、四爻，多主中途变化、互动压力。
动在五爻，多主关键节点或核心人物动作。
动在上爻，多主末端、迟滞、事后反馈。
```

### 禁止使用的表达

```text
明天一定有消息。
三天内必然改变。
某月某日一定发生。
对方一定会在某个具体时辰联系。
```

### 时间判断原则

- 动爻越低，事情越偏早期或暗中发动。
- 动爻居中，事情多在互动、压力、选择中变化。
- 动爻越高，事情越偏后期、收束或迟滞。
- 卦象静多动少，时间偏慢。
- 动爻多而杂，时间不稳，变化虽多但难落实。
- 本卦阻滞而变卦通达，表示先慢后动。
- 本卦通达而变卦阻滞，表示初期有动，后续变缓。

## Reality Check

卦象分析必须与现实信息并行判断。

六爻只能显示趋势和象意，不能否定已经发生的现实事实。

### 基本原则

- 现实事实优先于卦象推测。
- 已经发生的明确行为，不应被卦象重新解释成相反含义。
- 卦象中有感应，不等于现实中一定有行动。
- 卦象中有转机，不等于现实关系一定恢复。
- 卦象中有阻滞，不等于事情完全没有余地。
- 重大现实决策不能只依赖卦象。

### 示例

如果现实中对方已经明确拒绝，则不能因为卦象中有感应之象，就判断关系一定恢复。

正确表达：

```text
卦中有余气，但现实中已有明确拒绝，所以不能解读为关系即将恢复。更合理的判断是：情绪或记忆层面未必完全断绝，但现实行动层面仍然不宜推进。
```

错误表达：

```text
卦象显示她还想你，所以她一定会回来。
```

## Quality Standards

每次分析必须满足以下标准：

1. 明确说明卦象依据。
2. 区分当前状态、变化点和后续趋势。
3. 不把个人猜测伪装成卦象结论。
4. 不因用户期待某结果而迎合。
5. 对不确定处要明说。
6. 必须给出策略建议。
7. 多卦综合时必须提炼共同主线。
8. 时间判断只能给区间或倾向，不可绝对化。
9. 不使用情绪安抚作为主要内容。
10. 最后必须有一句话断语。
11. 每卦开始前必须确认具体问题与起卦时间。
12. 起卦时间至少精确到年月日时，缺时不作强断。
13. 每日最多三卦，超过后只复盘不新增。
14. 同一问题不可重复起卦，重复问题以前卦为准。
15. 卦象判断必须结合现实校验。

## Constraints

### 不应处理为绝对预测

六爻分析只能作为传统象数分析和策略参考，不应替代现实判断。

尤其涉及以下事项时，应谨慎：

- 医疗诊断
- 法律纠纷
- 投资决策
- 高风险人身安全
- 重大财务行为
- 考试录取、合同签署等现实结果

可以分析趋势，但必须提醒用户现实判断优先。

### 不应编造缺失信息

如果用户没有提供：

- 起卦时间
- 动爻
- 起卦方式
- 本卦
- 变卦

应基于已有信息分析，并说明假设。

不要假装已经排出完整六亲、六神、世应。

### 不应过度心理化

例如用户问：

```text
今天有人想起我吗？
```

可以分析：

- 卦象是否有应动
- 是否有感应
- 是否有回忆之象
- 是否有行动之象

但不要扩展成大段心理画像。

## Examples

### Example 1：单卦简断

用户输入：

```text
问题：我现在是否应该主动联系？
起卦时间：2026年6月15日19时30分
爻象：少阳、少阴、老阳、少阴、少阳、少阴
```

输出示例：

```markdown
## 卦象信息

问题：我现在是否应该主动联系
起卦时间：2026年6月15日19时30分
起卦方式：六爻爻象
爻序：按自下而上判断
动爻：三爻动
问题类型：策略型
判断强度：中

## 一、总体结论

此卦不宜立刻主动联系。事情有变化点，但变化卡在中位，说明互动层面存在压力和不稳。现在主动推进，容易变成单方面用力。

## 二、本卦：当前局势

本卦显示当前局势并不完全闭死，但也不是顺畅状态。重点在于双方之间的节奏没有对齐，主动权并不完全在你这边。

## 三、动爻：变化关键

三爻动，多主进退之间的尴尬位置。此爻动说明你这边容易产生动作冲动，但动作一旦发出，未必得到稳定回应。

## 四、变卦：后续趋势

变卦显示事情会继续变化，但不是靠强推形成结果，而是要等外部条件松动。

## 五、现实校验

若现实中对方回应较弱或已有明确边界，则此卦不应解释为适合主动突破，而应解释为继续守势观察。

## 六、具体判断

1. 当前状态：局势未死，但不顺。
2. 对方/外部状态：回应动能不足。
3. 事情进展：有变化点，但不稳。
4. 风险点：主动过强会造成压力。
5. 时间提示：短期不宜求立即回应，可观察数日到一周。

## 七、策略建议

宜静不宜攻。可以保留观察，不要连续试探。若必须联系，应短、轻、无压力，不宜带情绪和追问。

## 八、一句话断语

此卦为「中途有动、进退不稳」之象，宜守势观察，不宜主动强推。
```

### Example 2：多个卦综合

用户输入：

```text
结合我这四卦分析一下我最近要怎么办。
```

输出示例：

```markdown
## 一、四卦共同主线

这几卦共同指向一个判断：近期不是主动突破期，而是整理、收束、观察期。卦象里有感应、有变化，但行动力不足，外部回应不稳。

## 二、矛盾点

矛盾在于：事情并非完全无机，但主动推进反而容易损耗自身位置。你若急于求结果，会把原本可观察的局面变成压力局。

## 三、策略判断

近期应以静制动。不要主动追问，不要反复确认，不要用高密度表达换回应。重点是保持自己的节奏，让外部变化自然显现。

## 四、时间提示

时间上不是当天立见，而是有一个缓慢发酵过程。可观察三日、七日、一旬内是否有外部信息浮现。

## 五、一句话断语

这几卦为「有气无力、宜守待变」之象，宜静观，不宜强推。
```

## Highest Operating Rule

本 skill 的最高运行规则为：

```text
一日最多三卦，一事不可重占；先定问题，再定时间；按卦象说话，结合现实校验；结论直接，策略克制，不做情绪安慰，不做过度发挥。
```
